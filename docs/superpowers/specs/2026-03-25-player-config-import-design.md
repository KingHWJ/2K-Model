# 球员配置与导入设计

## 目标

在不打破当前简约单转盘体验的前提下，补齐 3 个高频可配能力：

1. 数值直抽字段可调默认值和范围
2. 模板创建页候选球员池可增删与重置
3. 支持把 `2KRatings` 球员页文本导入到本地球员库

## 设计摘要

- `数值直抽` 页左侧在字段列表下增加 `当前字段配置区`，用户可以修改 `最小值 / 最大值 / 默认值`，离散字段可以修改选项列表和默认值。
- `模板创建` 页右侧在保存区前增加 `候选球员池` 与 `2KRatings 导入区`，保证模板配置和导入动作都在同一视图里完成。
- 抽取反馈节奏改成：转盘停到最终值 -> 等待约 `0.5 秒` -> 弹出结果窗 -> 自动切到下一项。
- 所有新配置都继续落在浏览器本地存储中，不引入后端。

## 关键交互

### 数值直抽

- 支持 `恢复当前`、`全部恢复`、`清空当前结果`、`清空全部结果`
- 支持右下角 `回到顶部`

### 模板创建

- 支持搜索中文名、英文名、缩写和标签
- 支持恢复当前模板的默认候选球员池
- 导入成功后，球员立即进入本地球员库，可直接加入候选池

## 数据策略

- `numberFields` 从“固定 fallback 覆盖”改为“内置骨架 + 用户配置合并”
- `players` 允许写入用户导入的 `2KRatings` 球员资料
- `recommendedTemplates.candidatePlayerIds` 允许保存用户当前模板调整后的候选池
