# 转盘修正与数值直抽设计

## 目标

把当前创建流程里的转盘交互修正为真正的“饼图扇区停靠”逻辑，并新增一个独立的 `数值直抽` 标签页，让用户可以不依赖推荐模板，直接通过数字转盘为各个字段生成最终数值。

## 设计范围

本次设计只覆盖以下两部分：

1. `模板创建` 页的球员转盘修正
2. 新增 `数值直抽` 标签页

不在本次范围内的内容：

- 后端或云同步
- 导入导出
- Badge / Takeover / Build Specialization
- 自定义新增任意字段

## 导航结构

顶部导航调整为 5 个标签：

- 首页
- 模板创建
- 数值直抽
- 标签页
- 我的模板

其中：

- `首页` 仍用于推荐模板与封面上传
- `模板创建` 专注“按球员来源抽字段”
- `数值直抽` 专注“按数值范围直接抽字段”

## 模板创建页修正

### 交互目标

- 转盘必须是真正的饼图扇区
- 每一块扇区只代表一名球员
- 顶部指针固定并向下指
- 点击抽取后，指针最后只会停在某一块扇区上
- 停到谁，就取谁在当前字段上的真实值

### 页面结构

- 页面顶部：当前模板横幅、模板名、风格说明
- 页面中部：单个球员饼图转盘
- 页面旁侧：当前字段结果卡
- 页面底部或侧边：字段结果表

### 按钮布局

- 转盘下方保留一个主按钮：`开始抽取`
- 转盘右侧增加一个同功能按钮：`抽取当前字段`
- 保留 `返回首页`

### 结果写入方式

如果当前字段是 `身高`，转盘停在 `迈克尔·乔丹`，则：

- 最终值：`198cm`
- 来源球员：`迈克尔·乔丹`
- 备注：使用该球员元数据里的默认说明

并同步写入结果表：

- 字段
- 最终值
- 来源
- 备注

## 数值直抽标签页

### 定位

这是一个独立于推荐模板的创建工具。

特点：

- 不依赖模板候选球员
- 不依赖球员真实属性
- 通过数字转盘直接决定字段值

### 页面结构

- 左侧：字段面板
- 中间：数字饼图转盘
- 右侧：结果表与当前结果卡

### 字段面板

字段面板内置一组默认字段，每个字段显示：

- 字段名
- 默认值
- 范围
- 单位

点击字段后：

- 当前转盘切换到该字段的数字范围
- 当前结果卡显示该字段的默认信息

### 数字转盘

核心规则：

- 每个数字就是一个扇区
- 数字直接显示在扇区内部
- 顶部指针固定并向下
- 点击抽取后，转盘真实旋转
- 最后指针明确停在某一个数字扇区上

视觉要求：

- 使用浅色简约风
- 扇区边界清楚，但不过度刺眼
- 数字需要清晰，不做强烈倾斜和花哨描边
- 当前命中扇区高亮
- 在指针附近增加一个放大读数区，强化最终值识别

### 按钮布局

- 转盘下方主按钮：`开始抽取`
- 转盘右侧辅助按钮：`抽取当前字段`

## 默认字段与范围

### 身体尺寸

- 身高：默认 `198cm`，范围 `170-224cm`
- 体重：默认 `98kg`，范围 `70-170kg`
- 臂展：默认 `211cm`，范围 `175-250cm`
- 体型指数：默认 `5`，范围 `1-10`

说明：

- `体型指数` 作为“肩宽/体型”的表现形式，更贴近 Builder 概念
- 前端展示时可以映射为：`瘦长 / 标准 / 宽肩`

### 运动与技术属性

以下字段统一按能力值处理：

- 速度
- 加速
- 弹跳
- 力量
- 上篮
- 扣篮
- 近框终结
- 中投
- 三分
- 罚球
- 控球
- 传球
- 外防
- 内防
- 抢断
- 盖帽
- 后场篮板

默认能力值统一为：

- 默认值：`65`
- 范围：`25-99`

## 数据与状态设计

### 模板创建页

保留现有会话结构，但修正转盘状态：

- 当前指向球员
- 当前旋转角度
- 当前命中扇区

### 数值直抽页

新增独立状态：

- 当前字段 ID
- 当前字段范围
- 当前转盘角度
- 当前命中数字
- 每个字段已抽结果

该状态需要持久化到本地，避免刷新后丢失。

## 视觉方向

- 继续使用简约浅色基调
- 指针更像真实抽盘指针
- 扇区与数字更工整
- 不使用过强的霓虹色和重纹理
- 强调“清楚、稳、耐看”

## 测试方向

### 模板创建页

- 点击抽取后，当前字段能推进到下一项
- 结果表能正确写入真实字段值
- 转盘会把抽中的球员高亮显示

### 数值直抽页

- 切换字段后，数字范围能正确变化
- 点击抽取后，最终值必须落在字段范围内
- 结果表能保存当前字段的最终值
- 刷新后，本地状态能够恢复

## 参考依据

- `25-99` 能力值区间采用 2K 常见能力值概念
- 身高上限 `224cm` 参考 2K26 Builder 的公开信息
- 中文字段和标签为贴近式建模，不等于官方逐字字段表
