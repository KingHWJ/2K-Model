# 数据结构

## 本地存储键

- `2k26-fusion-builder:v1`

## 顶层状态 `AppState`

```ts
interface AppState {
  version: number
  presets: Preset[]
  categories: AttributeCategory[]
  players: PlayerProfile[]
  templates: SavedTemplate[]
  session: GenerationSession
}
```

## 预设池 `Preset`

```ts
interface Preset {
  id: string
  name: string
  description: string
  defaultCandidateCount: number
  availablePlayerIds: string[]
  availableCategoryIds: string[]
  tagLibrary: string[]
}
```

用途：

- 定义当前可抽的球员范围。
- 定义当前启用的属性组范围。
- 提供候选人数默认值和标签库。

## 球员档案 `PlayerProfile`

```ts
interface PlayerProfile {
  id: string
  name: string
  position: string
  overall: number
  era: string
  tags: string[]
  categories: Record<string, Record<string, number | string>>
}
```

用途：

- 保存单个球员的基础信息。
- 保存每个属性组下的具体字段值。

## 属性组 `AttributeCategory`

```ts
interface AttributeCategory {
  id: string
  name: string
  fields: Array<{
    key: string
    label: string
    defaultValue: number | string
  }>
}
```

用途：

- 定义工作台的属性大类。
- 控制球员编辑区和结果表格如何展示字段。

## 当前会话 `GenerationSession`

```ts
interface GenerationSession {
  presetId: string
  templateName: string
  candidateCount: number
  candidatePlayerIds: string[]
  assignments: Record<string, { playerId: string; createdAt: string }>
  drawLog: Array<{
    type: 'candidate' | 'category'
    targetId: string
    playerId: string
    createdAt: string
  }>
  createdAt: string
  updatedAt: string
}
```

用途：

- 记录当前工作台正在编辑的模板状态。
- 记录候选球员名单和各属性组来源球员。

## 已保存模板 `SavedTemplate`

```ts
interface SavedTemplate {
  id: string
  name: string
  presetId: string
  candidateCount: number
  candidatePlayerIds: string[]
  assignments: Record<string, { playerId: string; createdAt: string }>
  summary: {
    overall: number
    position: string
    tags: string[]
    categorySources: Array<{
      categoryId: string
      categoryName: string
      playerName: string
    }>
  }
  createdAt: string
  updatedAt: string
}
```

用途：

- 用于模板库展示和后续继续编辑。
- `summary` 是模板列表和右侧摘要的直接数据来源。
