# 2K26 球员融合模板工具

一个基于 `React + Vite + TypeScript` 构建的静态网页工具，用来从推荐模板出发，按 `身高 / 体重 / 臂展 / 肩宽 / 核心能力字段` 逐项抽取球员来源，快速拼出一套新的 2K26 风格模板。

## 当前版本亮点

- 首页先展示 `官方预设型模板推荐`，而不是直接进入复杂工作台。
- 点击 `基于此模板创建` 后，进入单个大转盘页面，按字段逐项抽取来源球员。
- 转盘会按真实目标扇区停靠，指针固定在顶部，盘面标签保持横向清晰可读。
- 创建模板页和数值直抽页都保留 `开始抽取` 与 `抽取当前字段` 两个紧凑入口，不再把长按钮堆在页面底部。
- 每次抽取后，转盘会先停在最终值上，延迟 `0.5 秒` 后弹出结果窗；只有手动关闭弹窗，才会进入下一字段。
- 支持独立的 `数值直抽` 标签页，所有内置字段都可以直接通过数字转盘抽出最终值。
- 新增独立的 `设置` 标签页，用来管理 `最小值 / 最大值 / 默认值 / 位置预设 / 2KRatings 导入`。
- `数值直抽` 支持按 `通用 / PG / SG / SF / PF / C` 位置预设切换身高、体重、臂展范围。
- `数值直抽` 支持 `清空结果 / 恢复当前默认 / 全部恢复默认`。
- `模板创建` 支持调整候选球员池，搜索并加入更多球员，也可以一键恢复模板默认候选池。
- 支持把 `2KRatings` 球员页文本粘贴进 `设置` 页，直接导入成新的球员资料。
- 支持 `标签页` 查看模板风格说明。
- 支持 `我的模板` 查看已保存结果，并继续编辑。
- 推荐模板卡支持 `系统默认封面` 与 `用户自定义上传封面`。
- 页面右下角提供 `回到顶部` 按钮，长页面操作更顺手。
- 数据全部保存在浏览器本地，无需后端。

## 本地启动

```bash
npm install
npm run dev -- --host 0.0.0.0
```

默认开发地址通常是 [http://localhost:5173](http://localhost:5173)。

## 说明与边界

- 当前默认球员数据优先满足“可演示、可玩、风格清晰”，不承诺与 NBA 2K26 游戏内字段逐项完全一致。
- 默认中文标签是基于 2K26 `Builder / archetype / scouting report` 思路做的贴近式建模，不是官方逐球员原文标签表。
- 数值直抽页内置了贴近 2K Builder 概念的默认范围，例如 `身高 170-224 cm`、`体重 70-159 kg`、`臂展 175-250 cm`，并把 `肩宽/体型` 改成 `精瘦 / 标准 / 宽肩 / 强壮` 这类更直观的离散选项；能力值默认基线为 `65`，范围为 `25-99`。
- 位置预设是工具侧的 `2K 风格范围预设`，用于模拟不同位置的大致尺寸约束，不是官方逐字段原文表。
- `2KRatings` 导入当前采用“粘贴球员页文本”的方式，避免静态网页直接跨域抓取站点时被浏览器限制。
- 默认封面使用内置海报风占位图，功能上支持你替换成自己的真实球星封面。
- `Badge / Takeover / Build Specialization / Cap Breakers` 暂未进入当前主流程，后续可以继续扩展。
